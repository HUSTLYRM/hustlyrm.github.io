template<std::default_initializable TItem>
class AsyncItemPool : public CodeHelper::DestinationNode<TItem>, public CodeHelper::SourceNode<TItem>#

用于异步存储和取值的数据池，理论能够保证不重复读取到最新数据； 这里有两个概念，reader 和 writer，分别是操作此数据池的角色，一般分布在两个不同的线程中。 reader 读取器，调用此类型对象的 AsyncItemPool::GetItem 函数获取数据，此调用可能失败； writer 写入器，调用此类型对象的 AsyncItemPool::SetItem 函数写入数据，此调用一定成功； 当且仅当 writer 写入速度大于 reader 读取速度时，GetItem 操作每次都能成功。

模板参数:

TItem – 数据池中存储的物品的类型

Public Functions

inline virtual void SetItem(const TItem &item) noexcept override#

向数据池中写入数据，正常情况下不阻塞当前线程，必定成功

警告

当有两个或以上的 writer 时，大概率会阻塞当前线程

inline virtual bool GetItem(TItem &item) noexcept override#

从数据池中获取数据，几乎不阻塞当前线程，可能会失败

返回:

当获取数据失败时，返回 false

class Counter#

计数器，指定最大值，计数到最大值后返回 true 否则每次计数时返回 false

备注

不再考虑有关线程安全的问题，也不推荐在多线程环境下使用 因为大多数使用场景不会计数的值太大，所以更换内部数据类型为 std::uint32_t

警告

在多线程环境下使用计数需自行加锁，且需要注意可能的数据溢出问题

Public Functions

Counter() noexcept#

构造计数器，设置最大值为 Counter::DefaultMaxCount

bool Reached() const noexcept#

检查是否到达了最大值 ( Counter::Current >= Counter::MaxCount )
