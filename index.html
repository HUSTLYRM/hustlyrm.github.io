template<std::default_initializable TItem>
class AsyncItemPool : public CodeHelper::DestinationNode<TItem>, public CodeHelper::SourceNode<TItem>#

用于异步存储和取值的数据池，用于不重复读取到较新的写入数据。 工作方式如下： 

      *---Pool--*
writer ->-+ SetItem | | GetItem +->- reader ——

此数据池在同一时间只支持一个写入器(writer)和一个读取器(reader)。 writer 写入器，调用此类型对象的 AsyncItemPool::SetItem 函数写入数据，一定成功； reader 读取器，调用此类型对象的 AsyncItemPool::GetItem 函数获取数据，可能失败； 当且仅当 writer 写入速度大于 reader 读取速度时，GetItem 操作才能每次都成功。

模板参数:

TItem – 数据池中存储的物品的类型

Public Functions

inline virtual void SetItem(const TItem &item) noexcept override#

向数据池中写入数据，正常情况下不阻塞当前线程，必定成功

警告

当有两个或以上的 writer 时，大概率会阻塞当前线程，直到有空位

inline virtual bool GetItem(TItem &item) noexcept override#

从数据池中获取数据，几乎不阻塞当前线程，可能会失败

返回:

当获取数据失败时，也就是没有任何已经准备好的数据时，返回 false

基于 PingPong 思想的运行时双缓冲内存区处理器 提供了处理缓冲并检查的方法，通过 Exchange 可以检查本次是否有完整的数据包， 如果没有则存储本次数据包等待下次数据包到来。

Attention

本类不保证线程安全，只要求内存区间的读写权限，且不负责销毁内存。

备注

目前承担了检验数据包的职责，待移除。

参数:

  • headByte – 头字节，用于检查是否是完整的数据包，不可以是 0 ，否则抛出 std::runtime_error

  • fullSpan – size 至少大于 2，否则会抛出 std::runtime_error

抛出:

std::span<std::uint8_t> GetReaderSpan() const noexcept#

流程.2 假设已经写入所有数据到读取缓冲区(即 Pong 缓冲区)，现在执行交换流程，并检查是否可以输出

参数:

  • destination – 用于输出数据的位置，大小必须大于或等于 GetReaderSpan().size()

  • crcVerify – 是否需要验证 CRC 校验码 (默认数据最后一位是 CRC8 校验码)

  • TMessage – 要写入的消息类型，只需要默认初始化

  • TWriter – 写入器的类型，需要提供一些信息便于日志记录

Public Functions

inline virtual bool Check() noexcept override#

用于检查任务的依赖状态，判断是否满足依赖，可以执行

返回:

如果任务可以执行，返回 true ，否则返回 false

inline virtual void Execute() noexcept override#

执行任务

inline virtual bool IsDone() const noexcept override#

用于判断任务是否已经执行完毕，要求判断的过程不对任务产生任何影响

返回:

只要任务执行完毕，即使是异常退出，都会返回 true ，其他情况会是 false

inline virtual void Interrupt() noexcept override#

要求中断任务，执行后可通过 IsDone 函数判断任务是否已经中断 任务一般通过在循环中检查此函数控制的一个标志量来提前中断任务，所以此函数不能保证立即中断任务

inline virtual void Reset() noexcept override#

要求重置任务状态，执行后可用过 IsDone 函数判断任务是否重置
